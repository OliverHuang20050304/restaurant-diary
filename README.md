# 我的餐廳日記 🍽️

一個優雅、簡潔的餐廳記錄網站，幫助你記錄每一餐的美好時光。

## ✨ 功能特色

### 使用者認證
- 🔐 **註冊功能**：建立個人帳號，保護你的餐廳記錄
- 🔑 **登入/登出**：安全的帳號管理系統
- 👤 **多使用者支援**：每個使用者擁有獨立的餐廳資料庫
- 🔒 **資料隔離**：不同使用者的資料完全分開儲存
- 🔐 **密碼保護**：使用 SHA-256 雜湊加密保護密碼（前端加密）

### 核心功能
- 📝 **餐廳記錄**：輕鬆新增、編輯、刪除餐廳資訊
- 🏷️ **分類系統**：支援中式、西式、日式、韓式、甜點、咖啡、其他等分類
- 🗺️ **Google Maps 整合**：一鍵開啟 Google Maps 查看餐廳位置
- ⭐ **評分系統**：1-5 星評分，記錄用餐體驗

### 進階功能
- 🔍 **搜尋功能**：快速搜尋餐廳名稱、地址或評論
- 🎯 **分類篩選**：依分類快速篩選餐廳
- ❤️ **收藏功能**：標記最愛的餐廳
- 📅 **造訪日期**：記錄用餐日期
- 💰 **價格標記**：標記價格範圍（$、$$、$$$、$$$$）
- 📝 **評論筆記**：記錄用餐心得、推薦菜色
- 📊 **統計資訊**：總餐廳數、分類數、平均評分、收藏數
- 📸 **照片支援**：可添加餐廳照片（網址）
- 💾 **資料持久化**：使用 localStorage 自動儲存資料

## 🎨 設計特色

- **優雅風格**：採用大地色系，簡約優雅的設計
- **響應式設計**：完美支援手機、平板、桌面裝置
- **流暢動畫**：平滑的過渡效果和互動體驗
- **直覺操作**：簡單易用的介面設計

## 🚀 使用方式

### 首次使用
1. 直接在瀏覽器開啟 `restaurant_diary.html`
2. 點擊「註冊」標籤建立新帳號
3. 輸入使用者名稱（3-20 個字元）和密碼（至少 6 個字元）
4. 註冊成功後自動跳轉到登入頁面
5. 使用剛建立的帳號登入

### 日常使用
1. 使用你的帳號登入系統
2. 點擊「+ 新增餐廳」開始記錄
3. 填寫餐廳資訊並儲存
4. 使用搜尋和篩選功能快速找到餐廳
5. 點擊「地圖」按鈕在 Google Maps 中查看位置
6. 點擊右上角「登出」可安全登出帳號

## 📋 資料儲存

所有資料都儲存在瀏覽器的 localStorage 中，無需後端伺服器。資料會自動保存，即使關閉瀏覽器也不會遺失。

### 儲存結構
- **使用者資料**：儲存在 `users` key 中，包含使用者名稱和加密的密碼
- **餐廳資料**：每個使用者的餐廳資料獨立儲存在 `restaurants_[username]` key 中
- **目前登入狀態**：儲存在 `currentUser` key 中

### 安全性說明
- 密碼使用 SHA-256 雜湊算法加密後儲存（前端加密）
- 每個使用者的資料完全獨立，無法互相存取
- ⚠️ **注意**：此為純前端應用，密碼加密僅為基本保護，不應儲存敏感資訊

## 🔧 技術規格

- **純前端**：HTML + CSS + JavaScript
- **無依賴**：不需要任何外部框架或庫
- **本地儲存**：使用 localStorage API
- **響應式設計**：CSS Grid 和 Flexbox

## 📝 注意事項

- 照片功能目前支援圖片網址，如需本地上傳功能需要後端支援
- Google Maps 連結會在新分頁開啟，無需 API Key
- 建議定期備份資料（可匯出 JSON 格式）
- **安全性提醒**：此為純前端應用，所有資料儲存在本地瀏覽器中
  - 不同使用者使用不同瀏覽器或清除 localStorage 後，資料會分開
  - 密碼僅使用前端加密，不適用於儲存高度敏感的資料
  - 建議僅用於個人或非敏感資訊的記錄
